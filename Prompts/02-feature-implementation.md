# Feature Implementation Prompt

Implement the following requirement using the existing codebase patterns and architecture.

Requirement:
[paste requirement]

Before making any code changes:

## 1. Requirement Analysis

* Summarize the requirement.
* Identify business rules.
* Identify validation rules.
* Identify assumptions.
* Highlight any unclear or missing information.

## 2. Existing Implementation Analysis

* Analyze the current implementation.
* Identify related modules, components, services, APIs, data models, and dependencies.
* Explain how the existing flow works.

## 3. Impact Analysis

* Identify impacted files.
* Identify impacted business flows.
* Identify potential regression areas.
* Identify backward compatibility concerns.

## 4. Edge Cases

Consider:

* User behavior edge cases.
* Data edge cases.
* State transition edge cases.
* Error scenarios.
* Concurrency or race condition scenarios (if applicable).

## 5. Implementation Plan

Provide:

* High-level approach.
* Step-by-step implementation plan.
* Files to modify.
* Reason for each modification.

After the analysis is complete:

## 6. Generate Code Changes

Implement the solution following existing project patterns.

Requirements:

* Reuse existing logic whenever possible.
* Follow existing naming conventions.
* Follow existing architecture and folder structure.
* Keep the solution maintainable and readable.
* Handle validation and error scenarios appropriately.
* Avoid unnecessary refactoring.

Do NOT:

* Change unrelated logic.
* Introduce breaking changes.
* Duplicate existing logic.
* Modify existing behavior unless required by the requirement.
