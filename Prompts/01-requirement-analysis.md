# Requirement Analysis Prompt

Analyze the following requirement.

Requirement:

[paste requirement]

Perform the following analysis.

---

## 1. Requirement Summary

Provide:

- Feature Purpose
- Business Objective
- Expected User Behavior

---

## 2. Functional Requirements

Identify:

- Explicit requirements
- Implicit requirements

For each requirement provide:

- Description
- Expected behavior

---

## 3. Business Rules

Identify:

- Explicit business rules
- Implicit business rules
- Hidden business logic

---

## 4. Validation Rules

Identify:

- Input validation
- Business validation
- State validation

For each validation provide:

- Validation rule
- Expected result

---

## 5. User Flow Analysis

Identify:

- Primary flow
- Alternate flows
- Error flows
- Permission flows

---

## 6. Edge Cases

Consider:

- User behavior edge cases
- Data edge cases
- Permission edge cases
- State transition edge cases
- Error scenarios

---

## 7. Requirement Gaps

Identify:

- Missing requirements
- Ambiguous requirements
- Assumptions

Explain:

- Why clarification is needed
- Risk if incorrect

---

## 8. Questions for BA / PO

Generate clarification questions.

Group by:

- Business rules
- Validation rules
- Permissions
- Error handling

Prioritize:

- High
- Medium
- Low

---

## 9. Acceptance Criteria

Generate acceptance criteria using:

Given
When
Then

Cover:

- Happy paths
- Validation scenarios
- Error scenarios

---

## 10. Risks

Identify:

- Functional risks
- Technical risks
- Regression risks

Provide:

- Impact
- Severity
- Mitigation

---

## 11. Suggested Test Scenarios

Generate:

- Happy path scenarios
- Validation scenarios
- Error scenarios
- Edge cases

---

## 12. Final Assessment

Provide:

### Requirement Completeness

- Complete
- Mostly Complete
- Incomplete

### Development Readiness

- Ready for Development
- Ready with Assumptions
- Requires Clarification

### Critical Open Questions

### Recommended Next Step
