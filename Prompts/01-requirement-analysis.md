# Requirement Analysis Prompt

Analyze the following requirement.

Requirement:
[paste requirement]

Perform the following analysis.

## 1. Requirement Summary

* Summarize the requirement in simple terms.
* Explain the business goal.

## 2. Functional Requirements

* List all functional requirements.
* Identify explicit and implicit requirements.

## 3. Business Rules

* List business rules.
* Identify hidden business logic.

## 4. Validation Rules

* Input validation.
* Business validation.
* State validation.

## 5. Edge Cases

Consider:

* User behavior edge cases.
* Data edge cases.
* Permission edge cases.
* State transition edge cases.
* Error scenarios.

## 6. Missing Information

Identify:

* Missing requirements.
* Ambiguous requirements.
* Assumptions being made.

## 7. Questions To Clarify

List questions that should be discussed with BA/PO.

## 8. Acceptance Criteria

Generate detailed acceptance criteria.

## 9. Risks

Identify:

* Functional risks.
* Technical risks.
* Regression risks.

## 10. Suggested Test Scenarios

Provide:

* Happy path scenarios.
* Validation scenarios.
* Error scenarios.
* Edge case scenarios.
