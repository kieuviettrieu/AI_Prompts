# Code Review Prompt

Review the implementation below.

Code:
[paste code or selected files]

Perform a thorough review.

## 1. Functional Correctness

Check:

- Requirement coverage.
- Business logic correctness.
- Missing scenarios.

## 2. Edge Cases

Check:

- User edge cases.
- Data edge cases.
- Error handling cases.

## 3. Code Quality

Review:

- Readability.
- Maintainability.
- Naming consistency.
- Reusability.

## 4. Performance

Check:

- Expensive operations.
- Unnecessary processing.
- Potential bottlenecks.

## 5. Security

Check:

- Validation.
- Authorization.
- Sensitive data handling.
- Common vulnerabilities.

## 6. Technical Debt

Identify:

- Code smells.
- Duplication.
- Tight coupling.
- Complexity issues.

## 7. Regression Risks

Identify:

- Areas likely to break.
- Side effects.

For every issue provide:

- Severity (Critical / Major / Minor)
- Description
- Why it matters
- Suggested fix

## 8. Overall Assessment

Provide:

- Strengths
- Weaknesses
- Recommended improvements
- Merge readiness assessment
